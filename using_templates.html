Separating Code

HTML templates are helpful because they allow programmers to separate HTML from other types of code, make code more readable, are easier to modify, and make code more secure through the use of escaping. Including HTML in python for example, makes code difficult to read and modify. Since HTML has to be written as a string or multiple strings in python, the code won’t be written with any indentation or highlighting which can make code very difficult to read and modify if the HTML is substantial. Also, auto escaping HTML with template engines like Jinja, makes it much easier to make sure that HTML escaping is functioning because the programmer doesn’t have to manually make sure that every area that should be escaped is escaped.

Avoiding Repetition

Avoiding repetition is important for programmers because it minimizes unnecessary use of code, makes code less prone to error, and makes code easier to correct when errors have been made. One way templates can help programmers avoid repetition is through template inheritance. For example, a website that has the same header at the top of each page could put the HTML for the header in one file and have the other pages inherit from it. This would avoid the repetition of writing the HTML for the header on every page and correcting it on every page if there was a need for a change. Another way templates can help programmers avoid repetition is through variable substitution. When typing HTML directly into python, it would require complicated string substitution to manipulate different values. However, if that same HTML is put into a template, variable substitution can be used with statements written into the HTML template that allow the programmer to manipulate different variables. Using templates therefore, makes adding functionality to HTML much more clear and effective.
